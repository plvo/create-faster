## What create-faster adds

A Wrangler-based deployment setup for the app. Hono already exports a Workers-compatible fetch handler (`export default app`), so no adapter is needed — local dev keeps running on Bun while Wrangler handles preview and deploy.

Files added:
```
wrangler.jsonc          # Worker config: name, main, compatibility_date, nodejs_compat
.dev.vars.example       # Local secrets template for `wrangler dev`
```

### wrangler.jsonc

```jsonc title="wrangler.jsonc"
{
  "$schema": "node_modules/wrangler/config-schema.json",
  "name": "my-api",
  "main": "src/index.ts",
  "compatibility_date": "2026-06-12",
  "compatibility_flags": ["nodejs_compat"]
}
```

The Worker `name` is the app name in turborepo mode and the project name in single mode.

**Scripts:**
- `deploy` - Deploy the Worker (`wrangler deploy`)
- `preview` - Run the Worker locally on the Workers runtime (`wrangler dev`)
- `cf-typegen` - Generate binding types (`wrangler types --env-interface CloudflareEnv cloudflare-env.d.ts`)

**Modified files:**
- `.gitignore` - Ignores `.wrangler/` (local state), `.dev.vars` (local secrets), and `cloudflare-env.d.ts` (generated types)

**Local secrets:**

`wrangler dev` reads secrets from `.dev.vars` (dotenv syntax). Copy `.dev.vars.example` to `.dev.vars` and fill in real values. Production secrets are managed with `wrangler secret put <KEY>`.

**Turborepo mode:**
`wrangler.jsonc` and `.dev.vars.example` are placed in `apps/<name>/`, and the Worker is named after the app. Run the scripts from the app directory.

**Integration notes:**
Cloudflare Workers and AWS Lambda are mutually exclusive Deploy libraries — an app can only deploy to one platform.
